# 4. Singular Value Decomposition (SVD)

## Why SVD Is Needed

Eigen decomposition only works for:

* Square matrices

[
n \times n
]

However, real-world datasets are often:

[
m \times n
]

where:

[
m \neq n
]

These are rectangular matrices.

---

## What SVD Does

SVD generalizes matrix decomposition to any matrix shape.

[
A = U\Sigma V^T
]

where:

* (U) = left singular vectors
* (\Sigma) = singular values
* (V^T) = right singular vectors

---

## Why It Matters

Most real-world machine learning datasets are rectangular.

Examples:

* Images
* User-item recommendation matrices
* Tabular datasets
* Text embeddings

SVD is one of the most important tools in machine learning.

---

## Practical Application

The lecture previews a demonstration where SVD is used to:

* Compress an image
* Reduce image size by 96%

while preserving most visual information.

---

## Basic SVD Example

    [5, 6]

# 5. Moore-Penrose Pseudoinverse

## Motivation

Ordinary matrix inversion requires:

1. Square matrix
2. Non-singular matrix

Many machine learning problems violate these conditions.

---

### Example

A dataset might have:

[
1000 \times 10
]

matrix dimensions.

This matrix cannot be inverted directly because it is not square.

---

## Solution

Use the Moore-Penrose Pseudoinverse.

Notation:

[
A^+
]

---

## Why It Matters

The pseudoinverse allows us to:

* Solve systems of equations
* Estimate unknown values
* Perform regression

even when matrices are not invertible.

---

## Practical Application

The lecture previews fitting a line to data points using the pseudoinverse.

This is the foundation of:

* Linear Regression
* Least Squares Optimization

---

## Example Code

-------------------code----------------------

    [1, 1],
    [2, 1],
    [3, 1]

# Singular Value Decomposition (SVD) — Structured Notes

## 1. Introduction to Singular Value Decomposition (SVD)

### What is SVD?

**Singular Value Decomposition (SVD)** is a matrix decomposition technique widely used in:

* Machine Learning
* Data Science
* Computer Vision
* Recommendation Systems
* Data Compression

Unlike **Eigen Decomposition**, which only works on **square matrices**, SVD works on **any real-valued matrix**, including rectangular matrices.

### Key Advantage

| Eigen Decomposition           | Singular Value Decomposition          |
| ----------------------------- | ------------------------------------- |
| Only square matrices          | Any real-valued matrix                |
| Uses eigenvectors/eigenvalues | Uses singular vectors/singular values |
| More restrictive              | More general                          |

---

# 2. SVD Formula

For a matrix:

[
A
]

Its Singular Value Decomposition is:

[
A = UDV^T
]

Where:

### U Matrix

[
U \in \mathbb{R}^{m \times m}
]

* Orthogonal matrix
* Columns are **Left Singular Vectors**
* Number of columns equals number of rows in A

---

### D Matrix

[
D \in \mathbb{R}^{m \times n}
]

* Diagonal matrix
* Contains singular values on the main diagonal
* All other entries are zero

---

### Vᵀ Matrix

[
V^T \in \mathbb{R}^{n \times n}
]

* Orthogonal matrix
* Rows are **Right Singular Vectors**
* Number of columns equals number of columns in A

---

# 3. Example Matrix

The lecture uses a non-square matrix:

[
A=
\begin{bmatrix}
1 & 2\
3 & 4\
5 & 6
\end{bmatrix}
]

Dimensions:

* Rows = 3
* Columns = 2

Therefore:

### U

[
3 \times 3
]

### D

[
3 \times 2
]

### V^T

[
2 \times 2
]

---

# 4. Computing SVD in NumPy

## Step 1: Create Matrix A

import numpy as np

A = np.array([
    [1, 2],
    [3, 4],
    [5, 6]

-------------------code----------------------

---

## Step 2: Compute SVD

-------------------code----------------------

-------------------code----------------------

### Returned Objects

#### U

-------------------code----------------------
print(U)
-------------------code----------------------

---

#### Singular Values

-------------------code----------------------

-------------------code----------------------

These are the singular values.

---

#### Vᵀ

-------------------code----------------------
print(VT)
-------------------code----------------------

Contains right singular vectors as rows.

---

# 5. Building the Diagonal Matrix D

The SVD function returns singular values as a vector.

We must place them along the diagonal.

## Step 1: Create Diagonal Matrix

-------------------code----------------------
D = np.diag(d)
-------------------code----------------------

Result:

-------------------code----------------------
[
 [9.52, 0],
 [0, 0.51]
]
-------------------code----------------------

But dimensions are:

[
2 \times 2
]

We need:

[
3 \times 2
]

---

## Step 2: Expand D

-------------------code----------------------
D = np.vstack([
    D,
    [0, 0]
])
-------------------code----------------------

Result:

-------------------code----------------------
[
 [9.52, 0],
 [0, 0.51],
 [0, 0]
]
-------------------code----------------------

Dimensions:

[
3 \times 2
]

Now matrix multiplication is possible.

---

# 6. Reconstructing the Original Matrix

After decomposition:

[
A = UDV^T
]

We can verify this.

## Step 1: Multiply D and Vᵀ

-------------------code----------------------
DV = np.dot(D, VT)
-------------------code----------------------

---

## Step 2: Multiply U and (DV)

-------------------code----------------------
A_reconstructed = np.dot(U, DV)
-------------------code----------------------

---

## Step 3: Display Result

-------------------code----------------------
print(A_reconstructed)
-------------------code----------------------

Output:

-------------------code----------------------
[
 [1. 2.]
 [3. 4.]
 [5. 6.]
]
-------------------code----------------------

The original matrix is recovered.

---

# 7. Understanding the Components

## Left Singular Vectors (U)

Columns of U.

Example:

-------------------code----------------------text
u1
u2
u3
-------------------code----------------------

Represent important directions in row space.

---

## Right Singular Vectors (V)

Columns of V.

Represent important directions in column space.

Since NumPy returns:

-------------------code----------------------
VT
-------------------code----------------------

the singular vectors appear as rows.

---

## Singular Values (D)

Diagonal entries:

-------------------code----------------------text
σ1
σ2
...
-------------------code----------------------

Measure the importance of each singular vector.

Larger singular values:

* Carry more information
* Capture more structure in the data

Smaller singular values:

* Carry less information
* Often removed during compression

---

# 8. Relationship Between SVD and Eigen Decomposition

SVD and Eigen Decomposition are closely related.

---

## Left Singular Vectors

The left singular vectors of A are the eigenvectors of:

[
AA^T
]

[
\text{Left Singular Vectors}
============================

\text{Eigenvectors}(AA^T)
]

---

## Right Singular Vectors

The right singular vectors of A are the eigenvectors of:

[
A^TA
]

[
\text{Right Singular Vectors}
=============================

\text{Eigenvectors}(A^TA)
]

---

## Singular Values

Non-zero singular values are:

[
\sigma_i
========

\sqrt{\lambda_i}
]

where:

[
\lambda_i
]

are eigenvalues of:

[
A^TA
]

or

[
AA^T
]

---

# 10. Important Notes

### Orthogonal Matrices

Both U and V are orthogonal:

[
U^TU = I
]

[
V^TV = I
]

---

### Singular Values Are Non-Negative

Unlike eigenvalues:

* Eigenvalues can be negative.
* Singular values are always ≥ 0.

---

### SVD Always Exists

For any real-valued matrix:

[
A
]

an SVD decomposition exists.

This makes SVD one of the most powerful decomposition techniques.

---

# 11. Best Practices

### Use NumPy's Built-in SVD

-------------------code----------------------
np.linalg.svd()
-------------------code----------------------

instead of implementing SVD manually.

---

### Verify Reconstruction

Always check:

-------------------code----------------------
U @ D @ VT
-------------------code----------------------

matches the original matrix.

---

### Understand Dimensions

For:

[
A_{m \times n}
]

Dimensions are:

| Matrix | Size  |
| ------ | ----- |
| U      | m × m |
| D      | m × n |
| Vᵀ     | n × n |

Dimension mismatches are a common source of errors.

---

### Keep Largest Singular Values

For compression and dimensionality reduction:

* Keep largest singular values.
* Discard small singular values.

This preserves most information while reducing storage.

---

# 12. Full Final Code

-------------------code----------------------
import numpy as np

# Create matrix
A = np.array([
    [1, 2],
    [3, 4],
    [5, 6]
])

# Compute SVD
U, d, VT = np.linalg.svd(A)

print("U:")
print(U)

print("\nSingular Values:")
print(d)

print("\nVT:")
print(VT)

# Create diagonal matrix
D = np.diag(d)

# Expand to match dimensions (3x2)
D = np.vstack([
    D,
    [0, 0]
])

print("\nD:")
print(D)

# Reconstruct matrix
A_reconstructed = U @ D @ VT

print("\nReconstructed A:")
print(A_reconstructed)

# Verify equality
print("\nOriginal A:")
print(A)
-------------------code----------------------

# 2. Why SVD Can Compress Images

Recall the SVD formula:

[
A = UDV^T
]

Where:

* (U) = Left singular vectors
* (D) = Singular values
* (V^T) = Right singular vectors

Not all singular values are equally important.

By convention:

* Singular values are ordered from largest to smallest.
* The first singular values capture most of the important information.
* Later singular values capture fine details and noise.

This allows us to approximate the image using only the first few singular values.

The reconstructed image is not identical to the original.

Increasing:

-------------------code----------------------
k
-------------------code----------------------

improves image quality.

But increases storage.

Smaller:

-------------------code----------------------

-------------------code----------------------

* Lower quality

Try:

k = 64
-------------------code----------------------

before increasing further.

# 19. Full Final Code

-------------------code----------------------
from PIL import Image
import numpy as np
import matplotlib.pyplot as plt

# Load image
img = Image.open("oboe-with-book.jpg")

# Convert to grayscale
img_gray = img.convert("LA")

# Convert to matrix
X = np.array(img_gray)[:, :, 0]

# Compute SVD
U, s, VT = np.linalg.svd(X)

# Reconstruction function
def reconstruct(U, s, VT, k):
    return U[:, :k] @ np.diag(s[:k]) @ VT[:k, :]

# Show compressed versions
for k in [2, 4, 8, 16, 32, 64]:
    recon = reconstruct(U, s, VT, k)

    plt.figure(figsize=(5,5))
    plt.imshow(recon, cmap="gray")
    plt.title(f"{k} Singular Values")
    plt.axis("off")
    plt.show()

# Compression statistics
rows, cols = X.shape

original_elements = rows * cols

k = 64

compressed_elements = (
    rows * k +
    k +
    cols * k
)

compression_ratio = (
    compressed_elements /
    original_elements
)

print("Original elements:", original_elements)
print("Compressed elements:", compressed_elements)
print("Percentage retained:", compression_ratio * 100)
print("Compression achieved:", (1 - compression_ratio) * 100)
-------------------code----------------------

*** 45. The Moore-Penrose Pseudoinverse ***
# Moore-Penrose Pseudoinverse

## 1. Overview

The **Moore-Penrose Pseudoinverse** is a linear algebra technique that allows us to work with matrices that cannot be inverted using ordinary matrix inversion.

It is especially important in **machine learning**, where systems of equations often involve:

* Non-square matrices
* Overdetermined systems
* Underdetermined systems
* Singular matrices

The pseudoinverse provides a way to solve these problems even when a regular inverse does not exist.

---

# 2. Why Regular Matrix Inversion Fails

A matrix can only be inverted when:

1. It is square

   * Number of rows = number of columns

2. It is non-singular

   * Columns are linearly independent
   * Determinant is not zero

If either condition fails, a regular inverse does not exist.

---

## 2.1 Overdetermined Systems

More equations than unknowns.

Example:

-------------------code----------------------
3 equations
2 unknowns
-------------------code----------------------

Matrix shape:

-------------------code----------------------
3 × 2
-------------------code----------------------

Problem:

* No single intersection point exists.
* Standard matrix inversion cannot be used.

---

## 2.2 Underdetermined Systems

Fewer equations than unknowns.

Example:

-------------------code----------------------
1 equation
2 unknowns
-------------------code----------------------

Matrix shape:

-------------------code----------------------
1 × 2
-------------------code----------------------

Problem:

* Infinite possible solutions.
* Standard inversion is impossible.

---

## 2.3 Singular Matrices

A square matrix can still fail to be invertible if its columns are linearly dependent.

Example:

-------------------code----------------------
Column 1 = [1, 2]
Column 2 = [2, 4]
-------------------code----------------------

Column 2 is simply:

-------------------code----------------------
2 × Column 1
-------------------code----------------------

This creates linear dependence.

Result:

-------------------code----------------------
det(A) = 0
-------------------code----------------------

The matrix is singular and cannot be inverted.

---

# 3. The Moore-Penrose Pseudoinverse

For a matrix:

-------------------code----------------------
A
-------------------code----------------------

The pseudoinverse is written as:

-------------------code----------------------
A⁺
-------------------code----------------------

Unlike the ordinary inverse:

-------------------code----------------------
A⁻¹
-------------------code----------------------

the pseudoinverse can be computed even when A is not square.

---

# 4. Pseudoinverse Formula

Using Singular Value Decomposition (SVD):

-------------------code----------------------
A = UDVᵀ
-------------------code----------------------

The pseudoinverse is:

-------------------code----------------------
A⁺ = V D⁺ Uᵀ
-------------------code----------------------

Where:

* U = left singular vectors
* V = right singular vectors
* D = diagonal matrix of singular values
* D⁺ = modified diagonal matrix

---

# 5. Step-by-Step Construction of the Pseudoinverse

Assume:

-------------------code----------------------
import numpy as np

A = np.array([
    [1, 2],
    [3, 4],
    [5, 6]
])
-------------------code----------------------

This matrix is:

-------------------code----------------------
3 × 2
-------------------code----------------------

and therefore not invertible.

---

## Step 1: Compute the SVD

-------------------code----------------------
U, d, VT = np.linalg.svd(A)
-------------------code----------------------

Results:

-------------------code----------------------
print(U)
print(d)
print(VT)
-------------------code----------------------

Outputs:

* U → left singular vectors
* d → singular values
* VT → right singular vectors (transposed)

---

## Step 2: Create Uᵀ

Transpose U:

-------------------code----------------------
UT = U.T
-------------------code----------------------

---

## Step 3: Create V

NumPy gives us:

-------------------code----------------------
VT
-------------------code----------------------

We need:

-------------------code----------------------
V
-------------------code----------------------

Transpose it:

-------------------code----------------------
V = VT.T
-------------------code----------------------

---

## Step 4: Create D⁺

The singular values are stored as a vector:

-------------------code----------------------
print(d)
-------------------code----------------------

Example:

-------------------code----------------------
[8.669 0.488]
-------------------code----------------------

Take the reciprocal of every non-zero singular value:

-------------------code----------------------
1 / 8.669
1 / 0.488
-------------------code----------------------

NumPy can do this automatically:

-------------------code----------------------
D_plus = np.linalg.inv(np.diag(d))
-------------------code----------------------

Result:

-------------------code----------------------
print(D_plus)
-------------------code----------------------

Example:

-------------------code----------------------
[[0.115 0.   ]
 [0.    2.048]]
-------------------code----------------------

---

## Step 5: Adjust Dimensions

To make matrix multiplication possible, we must match dimensions.

Add a column of zeros:

-------------------code----------------------
zeros = np.zeros((2, 1))

D_plus = np.concatenate(
    (D_plus, zeros),
    axis=1
)
-------------------code----------------------

Now D⁺ has the correct shape.

---

## Step 6: Calculate the Pseudoinverse

Apply the formula:

-------------------code----------------------
A_plus = V @ D_plus @ UT
-------------------code----------------------

Result:

-------------------code----------------------
print(A_plus)
-------------------code----------------------

This is the Moore-Penrose pseudoinverse.

---

# 6. Using NumPy's Built-In Method

NumPy already provides a direct implementation:

-------------------code----------------------
A_plus_builtin = np.linalg.pinv(A)
-------------------code----------------------

Display result:

-------------------code----------------------
print(A_plus_builtin)
-------------------code----------------------

The output matches the manually computed pseudoinverse.

---

# 7. Verifying the Result

Compare the two methods:

-------------------code----------------------
np.allclose(A_plus, A_plus_builtin)
-------------------code----------------------

Output:

-------------------code----------------------
True
-------------------code----------------------

This confirms:

-------------------code----------------------
Manual SVD approach = NumPy pinv()
-------------------code----------------------

---

# 8. Why the Pseudoinverse Matters in Machine Learning

The pseudoinverse is widely used because machine learning datasets often produce matrices that are:

* Non-square
* Overdetermined
* Underdetermined
* Not invertible

Applications include:

### Linear Regression

Finding the best-fit line.

### Least Squares Solutions

Solving systems with more equations than unknowns.

### Data Modeling

Estimating unknown parameters.

### Machine Learning Algorithms

Many optimization and fitting procedures rely on pseudoinverse calculations.

---

# 9. Relationship to SVD

Recall:

-------------------code----------------------
A = UDVᵀ
-------------------code----------------------

The pseudoinverse simply reuses the SVD components:

-------------------code----------------------
A⁺ = V D⁺ Uᵀ
-------------------code----------------------

Key difference:

| Matrix | Purpose                    |
| ------ | -------------------------- |
| D      | Original singular values   |
| D⁺     | Reciprocal singular values |

This shows why understanding **SVD** is essential before learning pseudoinverse.

---

### Always Use `pinv()` in Real Projects

Although it is useful to understand the mathematics:

-------------------code----------------------

-------------------code----------------------

is preferred in production code.

---

### Not Every Matrix Has an Inverse

But every real matrix has a pseudoinverse.

This is what makes pseudoinverse so powerful.

---

### SVD Is the Foundation

Understanding:

-------------------code----------------------
A = UDVᵀ
-------------------code----------------------

makes the pseudoinverse formula much easier to understand.

---

### Pseudoinverse Is Not the Same as Inverse

Regular inverse:

-------------------code----------------------
A⁻¹
-------------------code----------------------

Requires:

* Square matrix
* Non-singular matrix

Pseudoinverse:

-------------------code----------------------
A⁺
-------------------code----------------------

Works for:

* Square matrices
* Rectangular matrices
* Many non-invertible situations

*** 47. The Trace Operator ***
# Trace Operator

## 1. Overview

The **Trace Operator** is a simple but very useful concept in linear algebra.

It is commonly used to:

* Simplify matrix equations
* Rearrange mathematical expressions
* Work with matrix norms
* Derive machine learning algorithms such as PCA (Principal Component Analysis)

Notation:

[
\text{tr}(A)
]

where (A) is a matrix.

---

# 2. Definition of the Trace

The trace of a matrix is the **sum of the elements on the main diagonal**.

For a matrix:

[
A=
\begin{bmatrix}
25 & 2\
5 & 4
\end{bmatrix}
]

The diagonal elements are:

[
25,;4
]

Therefore:

[
\text{tr}(A)=25+4=29
]

---

# 3. Computing Trace in NumPy

## Step 1: Create a Matrix

-------------------code----------------------
import numpy as np

A = np.array([
    [25, 2],
    [5, 4]
])
-------------------code----------------------

---

## Step 2: Compute the Trace

### Manual Calculation

-------------------code----------------------
trace_manual = A[0,0] + A[1,1]
print(trace_manual)
-------------------code----------------------

Output:

-------------------code----------------------
29
-------------------code----------------------

---

### Using NumPy

-------------------code----------------------
trace_numpy = np.trace(A)
print(trace_numpy)
-------------------code----------------------

Output:

-------------------code----------------------
29
-------------------code----------------------

---

# 4. Trace Property #1

## Trace of a Matrix Equals Trace of Its Transpose

Property:

[
\text{tr}(A)=\text{tr}(A^T)
]

### Why?

Transposing a matrix swaps rows and columns.

However, the main diagonal remains unchanged.

Example:

[
A=
\begin{bmatrix}
25 & 2\
5 & 4
\end{bmatrix}
]

Transpose:

[
A^T=
\begin{bmatrix}
25 & 5\
2 & 4
\end{bmatrix}
]

The diagonal remains:

[
25,;4
]

So:

[
\text{tr}(A)=29
]

and

[
\text{tr}(A^T)=29
]

---

## Code Example

-------------------code----------------------
print(np.trace(A))
print(np.trace(A.T))
-------------------code----------------------

Output:

-------------------code----------------------
29
29
-------------------code----------------------

---

# 5. Trace Property #2

## Cyclic Property of Matrix Multiplication

Given three matrices:

[
A,;B,;C
]

The following traces are equal:

[
\text{tr}(ABC)
==============

# \text{tr}(CAB)

\text{tr}(BCA)
]

This property is called the **cyclic property of trace**.

---

## Why Is It Useful?

It allows us to rearrange matrix expressions into forms that are easier to analyze or differentiate.

This is extremely common in:

* Machine Learning
* Statistics
* Optimization
* Deep Learning derivations

---

## Example

-------------------code----------------------
A = np.random.randn(3, 3)
B = np.random.randn(3, 3)
C = np.random.randn(3, 3)

tr1 = np.trace(A @ B @ C)
tr2 = np.trace(C @ A @ B)
tr3 = np.trace(B @ C @ A)

print(tr1)
print(tr2)
print(tr3)
-------------------code----------------------

All values should be nearly identical.

---

# 6. Trace Property #3

## Frobenius Norm Using Trace

The **Frobenius Norm** is the matrix version of the vector L2 norm.

Formula:

[
||A||_F
=======

\sqrt{\text{tr}(A^T A)}
]

---

## Interpretation

1. Multiply:

[
A^T A
]

2. Take the trace

3. Take the square root

The result is the Frobenius norm.

---

# 7. Example Calculation

Given:

[
A=
\begin{bmatrix}
25 & 2\
5 & 4
\end{bmatrix}
]

### Compute

[
A^T A
]

-------------------code----------------------
AtA = A.T @ A
print(AtA)
-------------------code----------------------

---

### Take the Trace

-------------------code----------------------
trace_val = np.trace(AtA)
print(trace_val)
-------------------code----------------------

---

### Take the Square Root

-------------------code----------------------
fro_norm = np.sqrt(trace_val)
print(fro_norm)
-------------------code----------------------

---

-------------------code----------------------

-------------------code----------------------

# 10. Important Notes

### Trace Only Uses Diagonal Elements

Everything outside the main diagonal is ignored.

---

### Transpose Does Not Change Trace

Always true:

[
\text{tr}(A)=\text{tr}(A^T)
]

---

### Cyclic Rearrangement Is Allowed

Valid:

[
\text{tr}(ABC)
==============

# \text{tr}(BCA)

\text{tr}(CAB)
]

But arbitrary reordering is **not** allowed.

For example:

[
\text{tr}(ABC)
\neq
\text{tr}(ACB)
]

in general.

---

### Frobenius Norm Connection

Remember:

[
||A||_F
=======

\sqrt{\text{tr}(A^T A)}
]

This relationship appears frequently in:

* PCA
* Linear Regression
* Matrix Optimization
* Machine Learning loss functions

# 11. Full Final Code

-------------------code----------------------
import numpy as np
import torch

# --------------------------
# NumPy Example
# --------------------------

A = np.array([
    [25, 2],
    [5, 4]
])

# Trace
print("Trace:")
print(np.trace(A))

# Trace of transpose
print("\nTrace of A^T:")
print(np.trace(A.T))

# Frobenius norm via trace
fro_trace = np.sqrt(
    np.trace(A.T @ A)
)

print("\nFrobenius norm via trace:")
print(fro_trace)

# Frobenius norm via NumPy
fro_numpy = np.linalg.norm(A, 'fro')

print("\nFrobenius norm via NumPy:")
print(fro_numpy)

# --------------------------
# Cyclic Property Example
# --------------------------

A_rand = np.random.randn(3, 3)
B_rand = np.random.randn(3, 3)
C_rand = np.random.randn(3, 3)

tr1 = np.trace(A_rand @ B_rand @ C_rand)
tr2 = np.trace(C_rand @ A_rand @ B_rand)
tr3 = np.trace(B_rand @ C_rand @ A_rand)

print("\nCyclic Property:")
print(tr1)
print(tr2)
print(tr3)

# --------------------------
# PyTorch Example

A_torch = torch.tensor([
    [25., 2.],
    [5., 4.]

trace_torch = torch.trace(A_torch)

    )

# 12. Summary

The Trace Operator is defined as:

[
\text{tr}(A)=\sum_i A_{ii}
]

Key properties:

1. Trace is the sum of diagonal elements.
2. (\text{tr}(A)=\text{tr}(A^T))
3. (\text{tr}(ABC)=\text{tr}(BCA)=\text{tr}(CAB))
4. Frobenius norm can be computed using:

[
||A||_F
=======

\sqrt{\text{tr}(A^T A)}
]

Why it matters in Machine Learning:

* Simplifies matrix derivations
* Appears in optimization formulas
* Used in PCA and many matrix-based algorithms
* Provides a compact way to express matrix norms and objectives

The trace operator is simple to compute, but it becomes extremely powerful when working with advanced machine learning mathematics.
