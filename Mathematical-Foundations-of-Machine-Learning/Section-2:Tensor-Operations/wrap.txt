*** 15. Tensor Transposition ***
## 1. Introduction to Tensor Transposition

### 1.1 What is Transposition?

* **Transposition** is an operation that **rearranges the dimensions of a tensor**.
* The most common case:

  * Swap **rows and columns**

---

## 2. Transpose by Tensor Type

### 2.1 Scalar (0D Tensor)

#### Concept:

* A scalar has **no dimensions**
* Therefore, transposing it does **nothing**

[
x^T = x
]

---

#### Code Example:

-------------------code----------------------
x = 5
print(x)  # same after transpose
-------------------code----------------------

---

### 2.2 Vector (1D Tensor)

#### Concept:

* Transpose converts:

  * **Column vector → Row vector**
  * **Row vector → Column vector**

---

#### Example:

Column vector:

[
\begin{bmatrix}
1 \
2 \
3
\end{bmatrix}
]

Transpose:

[
\begin{bmatrix}
1 & 2 & 3
\end{bmatrix}
]

---

#### Code Example:

-------------------code----------------------
import numpy as np

v = np.array([[1],
              [2],
              [3]])

v_T = v.T

print(v)
print(v_T)
-------------------code----------------------

---

### 2.3 Matrix (2D Tensor)

#### Concept:

* Transpose flips matrix **over the main diagonal**
* Main diagonal:

  * From **top-left → bottom-right**

---

### 2.3.1 Rule

[
X_{i,j} \rightarrow X_{j,i}
]

* Row becomes column
* Column becomes row

---

### 2.3.2 Example

Original matrix:

[
\begin{bmatrix}
1 & 2 \
3 & 4
\end{bmatrix}
]

Transpose:

[
\begin{bmatrix}
1 & 3 \
2 & 4
\end{bmatrix}
]

---

### 2.3.3 Key Observations

* Shape changes:

  * (rows, cols) → (cols, rows)
* Elements remain the same
* Only positions change

---

*** 16. Basic Tensor Arithmetic, incl. the Hadamard Product ***
# 1. Introduction to Tensor Arithmetic

In this lecture, we learn how to perform **basic arithmetic operations on tensors** using:

* NumPy
* PyTorch
* TensorFlow

We focus on:

1. Scalar operations
2. Element-wise operations (Hadamard product)

---

# 2. Scalar Operations on Tensors

## 2.1 Concept

A **scalar operation** applies a single number to every element in a tensor.

### Examples:

* Multiply all elements by a number
* Add a constant to all elements

### Key Idea:

* The **shape of the tensor does not change**
* Operation is applied **element-wise with a scalar**

---

## 2.2 Step-by-Step Example (NumPy)

### Step 1: Create a Matrix

-------------------code----------------------
import numpy as np

X = np.array([
    [25, 2],
    [5, 26],
    [3, 7]
])
-------------------code----------------------

---

### Step 2: Multiply by Scalar

-------------------code----------------------
X_mul = X * 2
-------------------code----------------------

Result:

* Each element is multiplied by 2

---

### Step 3: Add Scalar

-------------------code----------------------
X_add = X + 2
-------------------code----------------------

Result:

* Each element increases by 2

---

### Step 4: Combine Operations

-------------------code----------------------
X_combined = X * 2 + 2
-------------------code----------------------

Important:

* Follows **order of operations** (multiply first, then add)

---

# 3. Scalar Operations in PyTorch and TensorFlow

## 3.1 PyTorch

-------------------code----------------------
import torch

X_torch = torch.tensor([
    [25, 2],
    [5, 26],
    [3, 7]
])

X_torch_result = X_torch * 2 + 2
-------------------code----------------------

---

## 3.2 TensorFlow

-------------------code----------------------
import tensorflow as tf

X_tf = tf.constant([
    [25, 2],
    [5, 26],
    [3, 7]
])

X_tf_result = X_tf * 2 + 2
-------------------code----------------------

---

## Important Note

* Operators like `+`, `*` are **overloaded**
* Internally:

  * PyTorch → `torch.add()`, `torch.mul()`
  * TensorFlow → `tf.add()`, `tf.multiply()`

---

# 4. Element-wise Operations (Hadamard Product)

## 4.1 Concept

The **Hadamard product** is element-wise multiplication between two tensors of the same shape.

### Requirements:

* Both tensors must have **same dimensions**

---

## 4.2 Step-by-Step Example (NumPy)

### Step 1: Create Two Matrices

-------------------code----------------------
A = X + 2
-------------------code----------------------

Now:

* `X` and `A` have the same shape

---

### Step 2: Element-wise Addition

-------------------code----------------------
add_result = X + A
-------------------code----------------------

Each element:

-------------------code----------------------
X[i][j] + A[i][j]
-------------------code----------------------

---

### Step 3: Hadamard Product (Element-wise Multiplication)

-------------------code----------------------
hadamard_result = X * A
-------------------code----------------------

Each element:

-------------------code----------------------
X[i][j] * A[i][j]
-------------------code----------------------

---

## 4.3 PyTorch Example

-------------------code----------------------
A_torch = X_torch + 2

add_torch = X_torch + A_torch
hadamard_torch = X_torch * A_torch
-------------------code----------------------

---

## 4.4 TensorFlow Example

-------------------code----------------------
A_tf = X_tf + 2

add_tf = X_tf + A_tf
hadamard_tf = X_tf * A_tf
-------------------code----------------------

---

# 5. Key Differences

| Operation Type          | Description                                    | Shape Change |
| ----------------------- | ---------------------------------------------- | ------------ |
| Scalar Operation        | Apply single number to all elements            | No           |
| Element-wise (Hadamard) | Operate between two tensors element-by-element | No           |

---

# 6. Important Notes & Best Practices

## 6.1 Shape Compatibility

* Element-wise operations require **same shape**
* Otherwise → error

## 6.2 Do Not Confuse with Matrix Multiplication

* `*` → Hadamard (element-wise)
* Matrix multiplication uses:

  * NumPy: `@` or `np.dot()`
  * PyTorch: `torch.matmul()`
  * TensorFlow: `tf.matmul()`

## 6.3 Use Vectorization

* Avoid loops
* Always prefer:

-------------------code----------------------
X * 2
-------------------code----------------------

instead of looping manually

## 6.4 Consistency Across Libraries

* NumPy, PyTorch, TensorFlow behave similarly
* Learn once → apply everywhere

---

import numpy as np

    ])

    print("Add 2:\n", X + 2)
    print("Combined:\n", X * 2 + 2)

    A = X + 2

        [25, 2],
        [5, 26],

    ])

    print("Add 2:\n", X + 2)
    print("Combined:\n", X * 2 + 2)

if __name__ == "__main__":

*** 17. Tensor Reduction ***
# 1. Introduction to Reduction

**Reduction** is a common tensor operation where we **combine multiple values into a single value**.

Most common example:

* **Sum of all elements in a tensor**

Reduction is widely used in:

* Machine learning (loss calculations)
* Statistics (mean, max, etc.)
* Data aggregation

---

# 2. Reduction on Vectors

## 2.1 Concept

For a vector:

[
x = [x_1, x_2, ..., x_n]
]

The reduced sum is:

[
\text{sum}(x) = x_1 + x_2 + ... + x_n
]

---

## 2.2 Example in NumPy

-------------------code----------------------
import numpy as np

x = np.array([1, 2, 3, 4])

total = x.sum()
print(total)  # Output: 10
-------------------code----------------------

---

# 3. Reduction on Matrices

For a matrix:

* You can sum **all elements**
* Or sum along a specific **axis (dimension)**

---

## 3.2 Example Matrix

    [25, 2],

-------------------code----------------------

* Others (mean, max) are also important but less frequent

*** 18. The Dot Product ***
# 1. Introduction to Dot Product

The **dot product** is one of the most important operations in linear algebra and machine learning.

It is used:

* In neural networks (every neuron uses it)
* In similarity calculations
* In projections and geometry

---

# 2. Definition of Dot Product

## 2.1 Requirements

To compute the dot product:

* Two vectors must have the **same length**

---

## 2.2 Mathematical Definition

Given two vectors:

[
x = [x_1, x_2, ..., x_n]
]
[
y = [y_1, y_2, ..., y_n]
]

The dot product is:

[
x \cdot y = x_1y_1 + x_2y_2 + ... + x_ny_n
]

---

## 2.3 Notation

Common notations:

* ( x \cdot y )
* ( x^T y )
* ( \langle x, y \rangle )

All represent the same operation.

---

# 3. Step-by-Step Calculation

## 3.1 Example Vectors

-------------------code----------------------
import numpy as np

x = np.array([25, 2, 5])
y = np.array([0, 1, 2])
-------------------code----------------------

---

## 3.2 Step 1: Element-wise Multiplication

-------------------code----------------------
products = x * y
print(products)  # [0, 2, 10]
-------------------code----------------------

---

## 3.3 Step 2: Sum the Results (Reduction)

-------------------code----------------------
dot_product = products.sum()
print(dot_product)  # 12
-------------------code----------------------

---

## 3.4 Combined Result

-------------------code----------------------
dot_product = np.sum(x * y)
-------------------code----------------------

---

# 4. Using Built-in Functions

## 4.1 NumPy

-------------------code----------------------
dot_product = np.dot(x, y)
print(dot_product)  # 12
-------------------code----------------------

---

## 4.2 PyTorch

-------------------code----------------------
import torch

x_torch = torch.tensor([25., 2., 5.])  # must be float
y_torch = torch.tensor([0., 1., 2.])

dot_product = torch.dot(x_torch, y_torch)
print(dot_product)  # 12.0
-------------------code----------------------

---

## 4.3 TensorFlow

-------------------code----------------------
import tensorflow as tf

x_tf = tf.constant([25, 2, 5], dtype=tf.float32)
y_tf = tf.constant([0, 1, 2], dtype=tf.float32)

# Step 1: element-wise multiply
products = tf.multiply(x_tf, y_tf)

# Step 2: sum
dot_product = tf.reduce_sum(products)

print(dot_product)  # 12.0
-------------------code----------------------

---

# 5. Key Intuition

The dot product is:

1. Multiply matching elements
2. Add everything together
3. Result is a **single scalar value**

---

# 6. Important Notes & Best Practices

## 6.1 Same Length is Required

* Vectors must have identical size
* Otherwise → error

---

## 6.2 Dot Product is a Reduction

* Final result is a **single number**
* Combines multiplication + summation

---

## 6.3 Data Types Matter (PyTorch / TensorFlow)

* Prefer using **float tensors**
* Some functions require floats (e.g., `torch.dot`)

---

## 6.4 Use Built-in Functions

Avoid manual implementation:

-------------------code----------------------
# Bad
sum(x[i] * y[i] for i in range(len(x)))

# 7. Full Final Code

-------------------code----------------------
import numpy as np
import torch
import tensorflow as tf


def numpy_demo():
    print("=== NumPy ===")
    x = np.array([25, 2, 5])
    y = np.array([0, 1, 2])

    # Manual computation
    products = x * y
    print("Element-wise product:", products)

    dot_manual = products.sum()
    print("Dot product (manual):", dot_manual)

    # Built-in
    dot_builtin = np.dot(x, y)
    print("Dot product (numpy):", dot_builtin)


def pytorch_demo():
    print("\n=== PyTorch ===")
    x = torch.tensor([25., 2., 5.])
    y = torch.tensor([0., 1., 2.])

    dot_product = torch.dot(x, y)
    print("Dot product (torch):", dot_product)


def tensorflow_demo():
    print("\n=== TensorFlow ===")
    x = tf.constant([25, 2, 5], dtype=tf.float32)
    y = tf.constant([0, 1, 2], dtype=tf.float32)

    # Step 1: multiply
    products = tf.multiply(x, y)

    # Step 2: reduce sum
    dot_product = tf.reduce_sum(products)

    print("Dot product (tensorflow):", dot_product)


if __name__ == "__main__":
    numpy_demo()
    pytorch_demo()
    tensorflow_demo()

# 2. Transpose of a Matrix

## 2.1 Concept

* Transpose flips a matrix over its **main diagonal** (top-left → bottom-right).
* Rows become columns, columns become rows.
* Element positions change:

  * Original: ( X_{i,j} )
  * Transposed: ( X_{j,i} )

## 2.2 Step-by-Step Example

Given matrix ( Y ):

-------------------code----------------------
Y =
[ a  b
  c  d ]
-------------------code----------------------

### Transpose:

-------------------code----------------------
Yᵀ =
[ a  c
  b  d ]
-------------------code----------------------

### Explanation:

1. First row → first column
2. Second row → second column
3. Order of elements stays the same

# 4. Dot Product of Vectors

## 4.1 Concept

* Works only on **vectors of same length**
* Steps:

  1. Multiply elements pairwise
  2. Sum the results

## 4.2 Step-by-Step Example

Given:

-------------------code----------------------

-------------------code----------------------

-------------------code----------------------

(0×2)  = 0
-------------------code----------------------

### Step 2: Sum

-------------------code----------------------
-5 + 20 + 0 = 15
-------------------code----------------------

### Final Result:

-------------------code----------------------
W · X = 15
-------------------code----------------------

# 6. Important Notes and Best Practices

## Transpose

* Scalars: unchanged
* Vectors: row ↔ column
* Matrices: flip across diagonal

## Hadamard Product

* Requires identical shapes
* Uses `*` operator in most libraries
* Not matrix multiplication

## Dot Product

* Vectors must have same length
* Result is always a **scalar**
* Widely used in machine learning (e.g., neural networks)

## General Tips

* Always check tensor **shape** before operations
* Understand difference between:

  * Element-wise operations
  * Matrix operations
* Use built-in functions (`dot`, `sum`, etc.) instead of manual loops

* **Transpose** flips tensor dimensions (rows ↔ columns)
* **Hadamard product** multiplies elements position-wise
* **Dot product** multiplies vectors element-wise then sums

These operations are fundamental in machine learning:

* Dot product → used in neural networks
* Hadamard → used in element-wise transformations
* Transpose → used in matrix algebra and optimization

Mastering these operations is essential before moving to advanced topics like matrix multiplication and linear systems.
